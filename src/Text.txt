## react intro

    What is React?

    1) React is a JavaScript library (not a framework) developed by Facebook (Meta) for building user interfaces (UIs), especially Single Page Applications (SPAs).

    2) It helps developers build reusable UI components that update efficiently when data changes.

    Instead of reloading a whole webpage when something changes, React updates only the parts of the UI that need to change.

    -------
    Why do we need React?

	It makes UI development faster, cleaner, and more efficient.

	Provides a better user experience with smooth updates without page reloads.

## useState() && form & Controlled input 

    useState is a react  hook that allows you to add state (data that can change over time)
    to a functional component.

    normally ,in js functionas, variable disappear after the function runs.
    but with useState, react remember value between re-render.

    useState is a React Hook that allows functional components to store 
    and manage data (state) and re-render the UI when that data changes.

    syntax:

        const[state,setState]=useState(initialvalue)

        state-- the curent value of the variable.
        setState-- a function that updates the  state.
        initialvalue -- the starting  value of  the state.

## useEffect()

        in React,useEffect is a hook that lets you perform side effect of a functional component
        Side effect are the task that happen outside the normal React rendering process,Like:

                Fetching data from API.
                Updating the browser DOM manually.
                Starting/stopinng timer.

        syntax:

            useEffect(()=>{
                //code...
            },[dependency 1,dependency 2.....])

            []-- if we doesn't mension any dependency mean's it will run only once at the begining.

## usecallBack ()

    usecallBack is a hook used to memoize (store) a function so that react does not recrate
    the function on every render unless dependency changes.It is mainly used for performance 
    optimization in react Applications.

    Every time  a component re-renders,
    It crecreate a brand new copy of every function inside it.

    Even if the function code looks exactly the same,
    React sees it as different because it's new object in memory.

    syntax:

        const name=usecallBack(()=>{
            // code
        },[dependency1, dependency2  ....])

## Routing in React && nested routing

    Routing means navigating  b/w different pages in a web app without refreshinng the  browser.
    In traditional websites ,clicking a link reloads the whole page.
    In react (SAP) ,routing lets us changes the view without reloading.

        Better performance (no full page reload)
        Smooth user Experiance
        Control over navigation without server request.


    commend:npm install react-router-dom

    structure:

        <BrowserRouter>
            <Routes>
                <Route path="adderss path" element={component}/>
                <Route path="" element={home}/>
                <Route path="" element={about}/>
            </Routes>
        </BrowserRouter>

        BrowserRouter:
            it is the main wrapper component for routing in react app.
            it enables navigation using the browser's URL (history URL) 
            Without this  ,routing will not working.

        Routers:
            It contains all the <Route/> component.
            It tells react router which component to show for which path.
            Only one rout will match and render at a time.

        Route:
            Define a specific URL path and which component should load  for that path.
